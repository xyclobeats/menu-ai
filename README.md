# 献立AI

ローカルで開くだけで使える、月間の献立表アプリです。材料、作り方、買い物リストを家族人数に合わせて表示します。

## できること

- 平日は朝ごはんと夕ごはんを自動作成
- 土日は朝・昼・夕ごはんを自動作成
- 土曜か日曜の夜に「とまりのお弁当」か「そそどのお弁当」を週1回組み込む
- 各メニューに主菜だけでなく副菜や汁物も付ける
- 各メニューの材料と作り方を確認できる
- 1週間ごとの必要食材リストを表示
- 1週間ごとの買い物金額の目安を表示

## 使い方

1. `index.html` をブラウザで開く
2. 年、月、子どもの人数、大人の人数、2種類のお弁当名を入れる
3. 「献立を作る」を押す
4. 右側の週間食材リストで、各週の買い物目安金額を確認する
5. メニューの `作り方を見る` から人数換算後の材料と手順を確認する
6. 必要なら各2週間分の `...を印刷` ボタンで、2週間ごとの買い物リストだけ印刷する

## 金額の考え方

- 食材ごとにざっくりした単価を持たせて、週単位で概算しています
- 土日のお弁当は、大人 700 円・子ども 500 円の目安で計算しています
- 地域、季節、購入先によって金額は前後します

## 人数換算の考え方

- 献立データの基準は `子ども2人・大人1人` です
- 子ども 1 人は大人 0.8 人分として換算します
- `卵 17 個` のような離散的な材料は切り上げます
- `キャベツ 2.5 玉` のように半分で扱いやすい材料は `0.5` 刻みで丸めます
- `少々` のような曖昧な表記は、多人数時に `小さじ` 表記へ寄せます

## GitHub Pages で公開する

1. このフォルダを GitHub の新しいリポジトリにアップロードする
2. デフォルトブランチを `main` にする
3. GitHub の `Settings > Pages` を開く
4. `Source` を `GitHub Actions` にする
5. `main` に push すると `.github/workflows/pages.yml` で自動公開される

公開URLの例:

- `https://ユーザー名.github.io/リポジトリ名/`
